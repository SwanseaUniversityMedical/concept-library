Concept Library API

The concept library API allows programmatic interaction with code lists, so that they can be directly used in queries, statistical scripts, etc.

- (Replace the number 0 in the url with the relevant concept/working set id.
Replace the number 1 in the url with the relevant version_id of the concept/working set.)

- (you can add the querystring ?format=json or ?format=xml to specify the required API output format.)

Concepts:

  1. To export the code list of the latest version of a concept

    2. {{ export_concept_codes }}

  2. To export the code list of a specific version of a concept

    3. {{ export_concept_codes_byVersionID }}

  3. To list and search for concepts

    4. {{ myConcepts }}

    - Lists all available concepts for the user
    - User can search with criteria using querystring parameters:
    (you can use one param or a combinations of them)

    • ?search=diab
      search by part of concept name (do not put wild characters here)
    • ?id=10
      you can specify concept id
    • ?tag_ids=11,4
      You can specify tags ids (get tags from /api/tags/)
    • ?show_only_my_concepts=1
      Only show concepts owned by me
    • ?show_deleted_concepts=1
      will show also deleted concepts (by default, deleted objects are not shown)
    • ?show_only_validated_concepts=1
      will show only validated concepts
    • ?brand=ADP
      will show only concepts with brand=ADP
    • ?author=moh
      search by part of the author name
    • ?owner_username=a.john
      search by full username of the owner
    • ?do_not_show_versions=1
      do not show concepts versions (by default, all concept version ids are shown)
    (Example: /api/myconcepts/?author=m&brand=adp&tag_ids=11)

  4. To list concept details for the latest version

    5. {{ myConceptdetail }}

  5. To list concept details for the specified version

    6. {{ myConceptdetail_version }}
